Planning de desarrollo para una plataforma de gestión de aspirantes por stands

Proyecto orientado a una empresa moderna que busca optimizar el registro de aspirantes mediante carga de Excel, trazabilidad por etapas y control operativo de stands como reclutamiento, selección, exámenes médicos, revisión de seguridad y contratación.

Visión del proyecto

  • Registrar aspirantes masivamente desde archivos Excel.
  • Controlar el flujo del candidato por cada stand del proceso.
  • Visualizar avance, estados pendientes y tiempos de atención.
  • Crear una base sólida para evolucionar a dashboards, alertas y analítica.

Proyecto base sugerido

Además del planning, se propone dejar definido el concepto inicial del software para poder continuar su construcción posteriormente.

Nombre tentativo

Talent Flow Hub


Objetivo

Centralizar el ingreso y seguimiento de aspirantes en jornadas masivas de selección, permitiendo que cada candidato avance por los distintos stands de forma controlada, rápida y medible.

Funcionalidades iniciales

  • Carga de aspirantes desde Excel.
  • Creación y administración de stands.
  • Cambio de estado del aspirante por etapa.
  • Consulta rápida del avance individual.
  • Panel de seguimiento operativo.
  • Base lista para escalar a reportería y automatización.

Planning de desarrollo

Estructura ajustada a un desarrollo ágil y realista, alineada con los tiempos estimados para construir un MVP funcional del sistema.

1
0.5 día

Ajuste del MER y definición de tablas

Se aterriza el modelo entidad-relación del proyecto y se definen las tablas necesarias para soportar el flujo operativo.

  • Revisión del proceso de candidatos y stands.
  • Definición de tablas principales y relaciones.
  • Normalización de campos clave.
  • Validación de estructura base para escalabilidad.
Entregable: MER ajustado y estructura de base de datos definida.
2
0.5 día

Setup base del proyecto

Se prepara la arquitectura inicial para dejar operativa la base técnica del sistema.

  • Configuración de FastAPI para backend.
  • Configuración inicial de React para frontend.
  • Conexión base a la base de datos.
  • Estructura inicial del repositorio y carpetas.
Entregable: proyecto base funcional con frontend, backend y BD conectados.
3
1 día

Login con roles y control de acceso

Se construye el acceso al sistema asegurando que cada perfil visualice y gestione solo lo que le corresponde.

  • Autenticación de usuarios.
  • Definición de roles y permisos.
  • Protección de rutas y módulos.
  • Control de acceso por perfil operativo.
Entregable: módulo de login y autorización por roles funcionando.
4
1 día

MFA con OTP

Se refuerza la seguridad del acceso agregando un segundo factor de autenticación mediante código OTP.

  • Generación y validación de OTP.
  • Integración del flujo MFA al login.
  • Manejo de expiración y reenvío.
  • Validación de experiencia de usuario segura y rápida.
Entregable: autenticación multifactor implementada.
5
1 día

Carga de candidatos desde Excel

Se desarrolla el módulo principal para importar aspirantes de forma masiva con reglas de validación.

  • Lectura del archivo Excel.
  • Validación de estructura y columnas esperadas.
  • Control de formatos y campos obligatorios.
  • Retroalimentación de errores de carga.
Entregable: carga masiva de candidatos funcional con validaciones iniciales.
6
1 día

Mapeo, persistencia y manejo de errores

Se transforma la información del archivo al modelo del sistema y se asegura una persistencia confiable.

  • Mapeo de columnas Excel a campos de base de datos.
  • Inserción y actualización controlada de registros.
  • Manejo de errores técnicos y funcionales.
  • Registro de inconsistencias para trazabilidad.
Entregable: proceso robusto de guardado de información con control de errores.
7
2 días

Control de duplicados, pruebas y ajustes

Se asegura la estabilidad del MVP validando la calidad de la información y ajustando el comportamiento del sistema.

  • Reglas para detección de candidatos duplicados.
  • Pruebas funcionales del flujo de carga.
  • Ajustes de validaciones y mensajes.
  • Corrección de incidencias encontradas en pruebas.
Entregable: MVP estabilizado, validado y listo para siguiente fase funcional.
